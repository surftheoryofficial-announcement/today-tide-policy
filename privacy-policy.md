# 개인정보처리방침

최종 업데이트: 2026년 7월 1일

본 개인정보처리방침은 `오늘의 간만조` / `Today's Tide` Android 앱과 iOS 앱 및
홈 화면 위젯에 적용됩니다.

## 1. 서비스 개요

`오늘의 간만조` / `Today's Tide`는 사용자가 선택한 해변의 조수 정보와
일출/일몰 정보를 확인하고, 홈 화면 위젯으로 조수 그래프를 볼 수 있도록
제공하는 모바일 앱입니다. Android에서는 Android 홈 화면 위젯을, iOS에서는
WidgetKit 홈 화면 위젯을 제공합니다.

조수 및 일출/일몰 정보는 참고용으로 제공됩니다. 항해, 구조, 잠수, 해상 작업,
위험 지역 출입 등 안전이 중요한 판단에는 공식 기관의 최신 안내와 현장 상황을
반드시 함께 확인해야 합니다.

## 2. 처리하는 정보

앱은 사용자 계정, 이름, 전화번호, 정확한 GPS 위치, 결제 정보를 직접 요구하거나
저장하지 않습니다.

앱에서 처리될 수 있는 정보는 다음과 같습니다.

| 항목 | 내용 | 보관/전송 |
|---|---|---|
| 선택한 해변 ID | 사용자가 앱에서 직접 선택한 해변을 기억하기 위한 내부 ID. 기기의 GPS 위치를 이용해 자동으로 결정하지 않습니다. | 사용자의 기기 내부 저장소에 저장 |
| 위젯별 해변 설정 | iOS WidgetKit 위젯이 앱 기본 해변과 다른 해변을 표시할 수 있도록 사용자가 위젯에서 선택한 해변 ID를 기억하기 위한 설정 | 사용자의 기기 내부 저장소에 저장. iOS는 App Group UserDefaults를 사용해 앱과 위젯이 공유할 수 있음 |
| 언어, 서퍼 모드, 차트 글자 크기 | 앱과 위젯 표시 방식을 유지하기 위한 설정 | 사용자의 기기 내부 저장소에 저장. iOS는 App Group UserDefaults를 사용해 앱과 위젯이 공유할 수 있음 |
| 캐시된 조석 데이터 | 앱과 위젯 그래프 표시, 오프라인/일시 실패 상태 대응, stale chart 방지 | 사용자의 기기 내부 저장소에 저장. iOS는 App Group 저장소를 통해 앱과 위젯이 공유할 수 있음 |
| 조수/일출/일몰 API 요청 정보 | 선택한 해변의 조수, 일출, 일몰 정보를 가져오기 위한 요청. 앱 또는 위젯 갱신 시 선택한 해변 ID, 대응 예보 지점, 조회 날짜 등이 전송될 수 있음 | 일반적인 인터넷 통신 과정에서 IP 주소, 요청 시각 등 기술 정보가 공공 데이터 API 제공자 또는 앱의 Firebase Functions API 프록시 서비스에서 처리될 수 있음 |
| 앱 사용 분석 정보 | 앱 실행, 해변 선택, 위젯 추가/삭제, 위젯 탭, 데이터 갱신 성공/실패 등 | Android와 iOS에서 Firebase Analytics가 활성화된 경우 처리될 수 있음. iOS는 `ANALYTICS_ENABLED` 설정과 `GoogleService-Info.plist`가 있는 빌드에서 Firebase Analytics를 사용할 수 있음 |
| 광고 관련 정보 | 광고 제공, 측정, 부정 이용 방지에 필요한 정보 | Android와 iOS 광고 활성 릴리스에서 Google Mobile Ads SDK 및 광고 파트너가 처리할 수 있음 |
| 문의 또는 의견 내용 | 사용자가 설정 화면의 의견 보내기 기능을 통해 외부 메일 앱에서 직접 작성한 이메일 주소와 문의 내용 | 사용자의 메일 앱을 통해 개발자 이메일로 전송될 수 있음 |

## 3. 분석 및 광고

Android 앱과 iOS 앱은 Firebase Analytics를 사용하여 기본적인 앱 사용 현황과
위젯 사용성을 파악하고, 데이터 갱신 안정성을 개선할 수 있습니다.

Firebase Analytics로 기록될 수 있는 이벤트 예시는 다음과 같습니다.

- 앱 실행
- 해변 선택
- 위젯 추가 및 삭제
- 위젯을 눌러 앱에 진입
- 조수 데이터 갱신 성공 또는 실패

이벤트에는 선택한 해변 ID, 지역명, Android 위젯 인스턴스 ID, 조수 데이터 날짜,
데이터 갱신 출처, 데이터 갱신 실패 유형 등이 포함될 수 있습니다.

Firebase Analytics 및 Google SDK는 분석 보고서, 품질 개선, 부정 이용 방지 등을
위해 기기 또는 앱 식별자, 앱 실행/상호작용 정보, 진단 정보, IP 주소 등 일반적인
기술 정보를 처리할 수 있습니다.

Android 앱과 iOS 앱에는 홈 화면 광고 영역이 있으며, 광고 활성 릴리스에서는
Google Mobile Ads SDK 및 광고 파트너가 광고 제공, 측정, 부정 이용 방지를 위해
정보를 처리할 수 있습니다. Android 릴리스는 Android Advertising ID 권한과 AdMob
application metadata를 포함할 수 있고, iOS 릴리스는 Google Mobile Ads SDK,
Google User Messaging Platform, `GADApplicationIdentifier`, `SKAdNetworkItems`를
포함할 수 있습니다. 실제 광고 및 분석 처리 범위는 Google Play Data safety 또는
App Store privacy labels에 맞춰 관리합니다. iOS에서 IDFA, AdSupport,
AppTrackingTransparency를 사용하는 광고 구성이 적용되는 경우 Apple의 App
Tracking Transparency 흐름에 따라 별도로 관리합니다. 현재 광고 송출 및 배포
범위에서는 EEA, UK, Switzerland 등 별도 광고 동의 요구 지역을 제외합니다. iOS 앱의
`AppAnalytics`는 `ANALYTICS_ENABLED` 설정과 Firebase 설정 파일이 있는 경우
Firebase Analytics로 이벤트를 전송하며, Firebase가 자동 수집하는 `app_open`
이벤트는 중복 전송하지 않습니다.

앱은 분석 이벤트 또는 광고 기능을 통해 사용자의 이름, 이메일 주소, 전화번호,
정확한 GPS 위치, 결제 정보, 계정 비밀번호를 수집하지 않습니다.

## 4. 직접 수집하지 않는 정보

앱은 다음 정보를 직접 요구하거나 앱 내부에 저장하지 않습니다.

- 이름
- 전화번호
- 계정 로그인 정보
- 사용자의 현재 위치 또는 정확한 GPS 위치
- 결제 정보
- 앱 내부 입력 폼에 저장되는 문의 또는 의견 내용

단, 사용자가 외부 메일 앱으로 의견을 보내는 경우 발신 이메일 주소와 문의 내용이
개발자에게 전달될 수 있습니다.

## 5. 정보 이용 목적

앱은 처리되는 정보를 다음 목적에 사용합니다.

- 사용자가 선택한 해변의 조수 그래프 표시
- 홈 화면 위젯의 조수 정보 표시 및 갱신
- 일출/일몰 정보 표시
- 언어, 서퍼 모드, 차트 글자 크기 등 사용자가 선택한 표시 설정 유지
- 데이터 갱신 성공률 및 실패 원인 파악
- 앱 기능 유지, 품질 개선, 오류 대응
- 사용자가 보낸 문의 또는 의견 확인, 답변, 앱 개선 참고
- Android/iOS 광고 활성 릴리스에서 광고 제공 및 광고 품질 관리

앱과 홈 화면 위젯은 사용자가 앱을 직접 열지 않은 상태에서도 최신 조수 정보를
표시하기 위해 정해진 시간 또는 시스템 갱신 시점에 네트워크 요청을 실행할 수
있습니다. 이 요청은 사용자가 앱에서 직접 선택한 해변의 조수/일출/일몰 정보를
가져오기 위한 기능 제공 목적이며, 정확한 GPS 위치 권한을 사용하지 않습니다.
iOS WidgetKit 위젯의 갱신 시점은 iOS 시스템 정책에 따라 결정됩니다.

## 6. 외부 서비스 및 SDK

앱 기능 제공을 위해 다음 외부 서비스 또는 SDK가 사용될 수 있습니다.

| 서비스 또는 SDK | 사용 목적 | 적용 범위 |
|---|---|---|
| 공공 데이터 API 또는 앱의 Firebase Functions API 프록시 서비스 | 조수 예보 및 일출/일몰 정보 조회, 앱/위젯 갱신 | Android, iOS |
| Firebase Analytics | 앱 사용 분석 및 품질 개선 | Android와 iOS. iOS는 `ANALYTICS_ENABLED` 설정과 `GoogleService-Info.plist`가 있는 빌드에서 활성화 |
| Google Mobile Ads SDK | 광고 제공, 측정, 부정 이용 방지 | Android와 iOS 광고 활성 빌드 |
| 사용자가 선택한 이메일 앱 및 이메일 서비스 | 사용자가 의견 보내기 기능으로 개발자에게 메일을 보내는 경우 문의 전달 | Android, iOS |

외부 서비스는 각 서비스의 개인정보 처리 방식과 정책에 따라 정보를 처리할 수
있습니다.

## 7. 보관 기간

- 선택한 해변 ID, 위젯별 해변 설정, 언어, 서퍼 모드, 차트 글자 크기, 캐시된
  조석 데이터는 사용자가 앱 데이터를 삭제하거나 앱을 제거할 때까지 기기에
  보관될 수 있습니다.
- Firebase Analytics 데이터는 Android 또는 iOS에서 Analytics가 활성화된 경우
  Firebase 및 Google Analytics의 데이터 보관 설정과 정책에 따라 처리됩니다.
- 개발자가 현재 별도 서버에 저장하는 사용자 계정 정보, 연락처 정보, 결제 정보는
  없습니다.
- 사용자가 의견 보내기 기능을 통해 개발자에게 메일을 보내는 경우, 이메일 주소와
  문의 내용은 문의 확인, 답변, 앱 개선 참고 목적에 필요한 기간 동안 개발자
  메일함에 보관될 수 있으며, 목적 달성 후 합리적인 기간 내 삭제합니다.

## 8. 사용자의 권리와 선택

사용자는 기기의 앱 설정에서 앱 데이터를 삭제하거나 앱을 제거하여 앱 내부에
저장된 선택 해변, 표시 설정, 캐시 데이터를 삭제할 수 있습니다.

광고가 표시되는 경우, 사용자는 Android 또는 Google 계정 설정, iOS 설정, 또는
Google/광고 파트너가 제공하는 선택 수단을 통해 광고 개인화 관련 설정을 변경할
수 있습니다. iOS에서 IDFA/App Tracking을 사용하는 광고 구성이 적용되는 경우
Apple의 App Tracking Transparency 흐름과 App Store privacy labels에 맞춰
관리합니다.

의견 보내기 기능으로 보낸 이메일 내용의 삭제를 원하는 경우, 아래 문의 이메일로
삭제를 요청할 수 있습니다.

## 9. 아동의 개인정보

앱은 아동을 주된 대상으로 하지 않으며, 이름, 연락처, 계정 정보, 정확한 위치
정보 등 아동의 개인정보를 직접 요구하지 않습니다.

## 10. 보안

앱은 필요한 범위의 정보만 처리하도록 설계합니다. 앱은 인터넷 권한을 사용하며,
정확한 위치 권한이나 연락처 권한 등 민감 권한을 요청하지 않습니다.

## 11. 방침 변경

앱 기능, 광고 설정, 분석 도구, 외부 서비스, 데이터 처리 방식이 변경되면 본
개인정보처리방침을 갱신합니다. 변경된 방침은 앱 배포 또는 공개된
개인정보처리방침 URL을 통해 안내합니다.
